# v1.4.0 Akari (2025-12-25)

光落终章，摇曳无声。

## 1. 新功能

### 1.1. 新的战绩卡片

在 1.4.0 版本中，重构了原有的战绩展示组件，使用了新的战绩卡片。

新的战绩卡片力求极致的信息罗列，提供了**事无巨细**的各类数据指标。

你可以体验如下全新的特性：

1. 适配各模式的总览表格式数据展示，包括队伍总击杀、金币、Ban 位和史诗级野怪击杀；玩家的六边形图、英雄、位置、海克斯、补兵和装备等。注意，部分列需要窗口更大的宽度以完全展示。

2. 详细到每一个条目的具体数据表格。甚至包括技能释放次数、信号发送次数、偷野和单杀次数等。更多细节可自行探索。

3. 现在可以查阅游戏内事件时间轴。它记录了每一次英雄击杀、建筑物摧毁、镀层获取。另外，可以查看每一次击杀时所在的地图位置，以及相关成员造成或受到的伤害明细。

4. 符文数据统计。现在可以查看所有人使用的符文统计数据，以分析不同符文的实际效果。你可以查看如本局“电刑”造成了多少伤害，或“公理秘术”提供了多少的冷却缩减和伤害提升。

5. 装备和技能构建已可查看。现在可以查看所有人的技能加点、装备购买路线，它们被按照时间详细罗列。另外，在斗魂竞技场和海克斯大乱斗中，League Akari 也会贴心地帮你统计每个玩家具体购买了几次锻造器。

6. 线图。正如其名，提供了基础的金币、补兵、经验时间线。以队伍或玩家为单位。

7. 对局回放下载。在同区可以下载本局回放。这是调起客户端的功能，因此过期的对局仍然不可观看。

> 详细数据高度依赖 SGP API，需开启 SGP 支持（默认）。

同步地，“对局”页面或其他可战绩预览的组件，都使用了这个新的战绩卡片。

### 1.2. 亮色模式支持

作为一个被搁置了很久的需求，现在它终于被实现了。现在，你可以通过设置项中的**亮色模式**选项，切换到亮色模式。

但在默认上，仍然使用暗色模式。这是因为客户端资源多采用暗色基调，实际上它在亮色模式下的表现并不尽如人意。

### 1.3. 更加完善的自动化选择工具

在之前的版本中，自动英雄选择、禁用和交换功能没有区分模式，且细节处理不够完善。

现在，这些功能已经完全重写。你可以按照模式、位置（排位队列）精准设置想要自动操作的英雄，或单独启用某一个模式系列的自动操作功能。

同时，部分条件下动作执行的时机已经调整。默认将采用 **预选** → **亮出** → **锁定** 的顺序流程，以让其更加自然。具体的时间间隔受设置项控制。

> 由于自动选择系功能的变动较大，原有的设置无法迁移到新版本。

### 1.4. 重构的召唤师搜索界面

在“战绩”主页面的标题栏，**搜索玩家** 功能得到了改善。现在已经完全放弃了原有的 UI 设计，采用面积更大的对话框进行展示。

现在将近期访问和好友列表放在的单独的左侧，同时提供了一个合区参照，以快速查阅所在大区位于哪个联合服务器。

在搜索玩家时，将允许使用 SGP API，优化了搜索逻辑，搜索速度最多提升 1500%。精确按照名称和 tag 搜索，将自动查询其他可用大区。

### 1.5. 房间内的对局分析

在设置 → 对局分析 → 在房间阶段生效 中，可开启这个功能。

开启后，League Akari 会查询所有位于房间中玩家的战绩或其他数据，就像位于英雄选择阶段或游戏中一样。

### 1.6. 卸载 League Akari

在设置 → 应用 → 卸载应用 中，提供了一个卸载途径。这将彻底删除 League Akari 以及其所有数据，不可恢复。

### 1.7. 其他次要更新

参见”调整“一节。

## 2. 调整

在提供新功能的同时，我们对软件内部的一些功能也进行了调整。具体名单包括但不限于如下。

- 背景材质支持 Mica（仅 Windows 11）。
- 游戏流 → 自动重连功能，现在提升了重连延时（具体来说，是 10 秒），以避开断线后第一时间无法重连的时间段。
- OP.GG 窗口，在设置中提供了一个永久关闭此窗口的选项。它和主设置中的对应设置项完全等同，仅作方便用户操作。
- OP.GG 窗口，现在会在合适的时机自动清理之前创建的装备方案。
- 在未连接或未创建战绩页时，提供一个启动页面。
- 部分设置项将可同时加载本地预设，或从远程仓库同步更新的数据。
- 有关战绩页面和对局分析中 SGP 和 LCU API 的设置，已经合并并移动到设置项中的 应用 → 首选数据源 选项中。默认是 SGP。
- 提供了一个新的功能，游戏流 → 发送地图位置信息。当处于极地大乱斗、海克斯大乱斗或无限乱斗时，可提前告知己方位于红方/蓝方。
- 基于推断的预组队检测功能已回归。但是现在的阈值默认为 5 场，以降低误差。
- 第一次启动的免责声明现在提供了快速关闭的方式。
- 在“自动化”的“其他”页面中，添加了一个功能。以预约邀请某些好友。这些好友将在可被邀请时被立即邀请。
- 为好友工具添加了一个简单的搜索筛选功能。
- 现在跨区战绩查询，不可查询段位将显式提示。
- 提供了更多的异常检测机制，包括 SGP 数据源连接不可用，或无法查询客户端命令行。
- 提供了一个内容保护开关，位于设置 → 其他 → 内容保护中。开启后，League Akari 将不会出现在屏幕捕获工具中。（它会影响 NVIDIA 的录制功能，但不会影响如 OBS 等其他工具）
- 对局分析页面的玩家卡片会自动在一定范围内调整高度。
- 观战功能调整。由于 Riot 更改了观战系统，因此无法实现任意玩家的观战了。但是在某些服务器，它仍然可用，只是高度依赖 SGP API。
- “自动英雄配置”不再会和 OP.GG 的自动设置冲突。
- 由于微软已经逐渐在新版中废除 WMIC，因此客户端命令行备选方案从 WMIC 迁移到 WMI API。
- 新增了信息补全特性。League Akari 会在英雄选择阶段或对局进行阶段中额外收集一些数据，补全缺失的信息。

## 3. BUG 修复

同时，一些 BUG 也被修复了。

- 缺失的 i18n 条目已经补充完毕。不会出现如 `auto-champ-select.delayed-ban` 等原始文本。
- 一些设置项无法跨窗口被同步。现在会被正确同步。
- 观战情况状态，不会错误地记录玩家信息。
- 闪现异位的 D 和 F 是反的，现已纠正。
- 之前调用 WMIC 出现错误时，会导致整个应用崩溃。已经妥善处理。

暂未修复的问题，将在之后的小型版本中（v1.4.1+）逐渐解决。

# v1.3.0 Sakurako (2024-12-05)

## 协议变更

在之前的版本中，League Akari 采用了 **MIT** 开源协议。然而，我们发现部分开发者对软件进行直接套壳，并将其用于商业用途，虽然这仍然符合 MIT 协议，但与 League Akari 的开发初衷相悖。

v1.3.0 版本对底层代码进行了重构，同时，自本版本起，League Akari 的开源协议变更为 **GPL-3.0**。

- English is now supported. You can switch the language in the settings.

- 现在同大区内的玩家标记将不再隔离。[#94](https://github.com/Hanxven/LeagueAkari/issues/94)

- 战绩页面的右键菜单中添加截图选项。

- 添加了一个工具，可以锁定设置项配置文件。 [#58](https://github.com/Hanxven/LeagueAkari/issues/58)

- 现在自动选择和自动禁用添加了实验性的延时设置，它的时间并不一定准确。[#93](https://github.com/Hanxven/LeagueAkari/issues/93) [#142](https://github.com/Hanxven/LeagueAkari/issues/142)
